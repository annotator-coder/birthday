## 4. 3-레이어 DAX 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│  LAYER 3 · 에이전트 레이어 (12개월 목표)                   │
│  모니터링 Agent · 초안생성 Agent · 품질검토 Agent          │
│  배포 Agent · 성과분석 Agent · 아카이빙 Agent             │
└─────────────────────────────────────────────────────────┘
         ↑ 에이전트가 실행, 사람이 감독·승인
┌─────────────────────────────────────────────────────────┐
│  LAYER 2 · 프로세스 레이어 (3개월 목표)                    │
│  업무별 표준 SOP · AI 프롬프트 라이브러리                  │
│  템플릿 체계 · 리뷰 프로세스 표준화                        │
└─────────────────────────────────────────────────────────┘
         ↑ AI가 업무 흐름에 내재화
┌─────────────────────────────────────────────────────────┐
│  LAYER 1 · 도구 레이어 (즉시 시작)                         │
│  Claude / ChatGPT · Perplexity · Canva AI · Notion AI   │
│  ElevenLabs · Gamma · Google Workspace AI               │
└─────────────────────────────────────────────────────────┘
```

**핵심 원칙:**
1. 레이어는 아래서 위로 순차 구축 (1→2→3)
2. 각 레이어는 독립적으로 가치 제공
3. 사람의 판단·승인 권한은 항상 유지
4. 홍보 특성상 톤·브랜드 일관성 사람이 최종 검수

## 6. Core-Derivative 콘텐츠 파이프라인

DAX 자동화의 중심축. 콘텐츠 1개(Core)를 6개 채널로 자동 파생하여 동일 팀·예산으로 도달 5배 달성.

### Core 제작 Agent 파이프라인 (이미 구축 중)
```
기획 Agent → 리서치 Agent → 초안 작성 Agent → 팩트체크 → 브랜드보이스 → SEO → 👤 최종검토
```

### Derivative 자동 파생
| 채널 | 타이밍 | AI 도구 | 산출물 |
|------|--------|---------|--------|
| LinkedIn | Day 0 | Claude + 임원 보이스 프롬프트 | 임원 인사이트 포스트 200자 |
| Instagram | Day 1 | Claude + Canva AI | 카드뉴스 5장 |
| YouTube | Day 3 | Claude + ElevenLabs | 숏터뷰 스크립트·자막 |
| 뉴스레터 | Week 2 | Claude + Stibee | 구독자 큐레이션 발송본 |
| Magazine | Monthly | Claude + Gamma | 영상 시리즈 기반 매거진 원고 |
| 홈페이지 Hub | Auto | Claude API + CMS | 뉴스룸 자동 발행 + SEO 메타 |

## 9. 4단계 DAX 마스터 플랜

### Phase 1 · 지금~1개월: 전 도메인 AI 도입
**목표:** 팀원 모두가 AI를 쓰기 시작. 도메인별 Quick Win 경험.

도메인별 실행:
- **대외홍보:** Claude로 보도자료 초안 → 위기등급별 프롬프트 매뉴얼화 (기구축 고도화)
- **SNS:** Canva AI + Claude 캡션 → 브랜드 보이스 프롬프트 완성 (기구축 고도화)
- **Internal Comm:** G식노트·언박싱·Face+Time 스크립트 프롬프트 1개씩 + Magazine 초안 프롬프트
- **PI:** 과거 연설문 → Claude Project RAG 구축
- **CSR:** 대외홍보 Agent 공유 + CSR 특화 템플릿
- **Stakeholder:** VM 성과 취합 자동화 + 보고서 초안 프롬프트
- **스포츠:** 경기 결과 → SNS 콘텐츠 자동 초안
- **캠페인:** 기획안 초안·카피 생성 프롬프트

**성공 기준:** 팀원 7명 전원이 AI 도구를 주 3회 이상 사용. 도메인별 "AI 덕분에 OO가 쉬워졌다" 사례 1건씩 수집.

---

### Phase 2 · 2~3개월: SOP 표준화 (B 레벨 달성)
**목표:** AI 사용을 팀의 공식 업무 방식으로 제도화.

산출물:
- 도메인별 AI 사용 가이드 1장 문서
- 공용 프롬프트 라이브러리 (Notion)
- 콘텐츠 브리프 템플릿 체계
- 검토 체크리스트 (브랜드보이스·팩트)
- AI 초안 → 팀원 편집 → 팀장 승인 표준 게이트
- Executive Voice 파일럿 (임원 1인 시범)
- Policy Watch 뉴스레터 1호 발행

**성공 기준:** 팀원이 SOP 없이도 자연스럽게 AI 사용. 산출물 제작 시간 평균 30% 단축. 팀장이 전략 판단에 집중 가능한 구조 완성.

---

### Phase 3 · 4~6개월: Core-Derivative 파이프라인 (C 레벨 진입)
**목표:** Phase 1·2에서 AI 효능 체득 후 콘텐츠 생산 자동화 파이프라인 가동.

실행:
- Core 제작 Agent 파이프라인 통합 완성
- Derivative 자동 파생 구현 (6채널 동시 발행)
- YouTube 에버그린 시리즈 첫 6편 제작·발행
- LinkedIn 임원 인사이트 정규 운영
- Partner Story 첫 편 제작
- 에너지 인사이트 팟캐스트 시범 운영
- 전 팀원 AI SOP 내재화 완료

**성공 기준:** 콘텐츠 1건 제작 시간 50% 단축. 뉴스레터 구독자 1,000명. YouTube 에버그린 6편 발행.

---

### Phase 4 · 7~12개월: 생태계 자동화 (C 레벨 완성)
**목표:** 新홈페이지 Hub 전면 가동 + 전 도메인 에이전트 통합.

홈페이지 Hub 5대 요소:
1. Newsroom 섹션 (TotalEnergies 모델) — 보도자료·인터뷰·ESG 리포트 통합
2. 구독 CTA 상시 노출 — Owned Audience 축적
3. 콘텐츠 시리즈 독립 URL — SEO 에버그린 자산
4. 멀티미디어 Kit — 고해상도 Photo·Video Kit 기본 제공
5. Energy Data Story — 인터랙티브 인포그래픽

기술 스택:
- Claude API + Make.com/n8n (워크플로우 자동화)
- Notion (지식베이스·아카이빙)
- Slack/Teams (알림 허브)
- Google Analytics (성과 피드백)

**12개월 KPI:**
- 콘텐츠 도달 5배 (Core-Derivative 운영)
- 뉴스레터 구독자 1만명
- YouTube/Instagram 구독자 1만명
- 검색 유입 비율 30%+
- 국내 에너지 미디어 1위 포지셔닝

## 10. 거버넌스 원칙

1. **사람 우선 게이트:** 모든 외부 발행 콘텐츠는 사람의 최종 승인 필수
2. **브랜드 보이스 책임:** AI 초안의 톤·뉘앙스 최종 검수는 팀원 담당
3. **사실 검증 의무:** AI 생성 팩트는 원본 소스 대조 확인 후 발행
4. **지식 축적:** 모든 산출물은 완료 즉시 Notion 지식베이스에 등록
5. **학습 루프:** 월 1회 팀 회고 — AI 효과 측정·프롬프트 개선·사례 공유
